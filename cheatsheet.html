Basic Functions

Here are some of the most commonly used functions in SpiLib for 2D and 3D game development:

  • spilib_init(width, height, title)

    Initializes the game window with specified width, height, and title. It sets up the framebuffer and GLFW window.

  • spilib_begin_frame()

    Starts a new frame by clearing the framebuffer to prepare for rendering. This should be called at the beginning of each game loop.

  • spilib_end_frame()

    Ends the current frame, draws the framebuffer to the screen, swaps buffers, and processes window events.

  • spilib_draw_pixel(x, y, color)

    Draws a single pixel on the screen at the specified (x, y) coordinates with the given color.

  • spilib_draw_rectangle(x, y, width, height, color)

    Draws a filled rectangle at the specified (x, y) coordinates with the given width, height, and color.

  • spilib_key_pressed(key)

    Checks if a specific key (e.g., GLFW_KEY_LEFT) is pressed during the current frame.

  • spilib_mouse_pressed(button)

    Checks if a specific mouse button (e.g., GLFW_MOUSE_BUTTON_LEFT) is pressed during the current frame.

  • spilib_is_running()

    Returns true if the window should remain open, or false if it should close. Typically used in the game loop to determine if the game is still running.

Advanced 3D Functions

For 3D games, SpiLib provides a few powerful functions for rendering 3D objects:

  • spilib_matrix_rotate(angle)

    Creates a rotation matrix to rotate objects by the given angle (in degrees) around the Z-axis.

  • spilib_draw_cube(x, y, z, size, angle)

    Draws a 3D cube at the given (x, y, z) coordinates with a specified size and rotation angle.

  • spilib_draw_voxel(x, y, z, color)

    Draws a voxel at the specified (x, y, z) coordinates with the given color. This is useful for creating block-based worlds like Minecraft.

Physics and Collision Detection

SpiLib includes basic physics functions to simulate gravity and check for collisions:

  • spilib_apply_gravity(&y, gravity)

    Applies gravity to an object's vertical position (y). The gravity parameter is the acceleration applied per frame.

  • spilib_check_collision(x1, y1, w1, h1, x2, y2, w2, h2)

    Checks if two rectangles, defined by their (x, y) position and (w, h) width and height, are colliding.

Camera and Movement

The following functions handle the camera's movement and rotation in a 3D world:

  • spilib_camera_create(x, y, z, rotation)

    Creates a camera at the specified position (x, y, z) and initial rotation angle.

  • spilib_camera_move(&camera, dx, dy, dz)

    Moves the camera by the specified amounts (dx, dy, dz). Useful for translating the camera in the 3D space.

  • spilib_camera_rotate(&camera, d_angle)

    Rotates the camera by the given angle (d_angle) in degrees.

Multiplayer and Networking

SpiLib can be used to create multiplayer games. Below are the basic functions for networking:

  • spilib_multiplayer_init()

    Initializes basic networking for multiplayer games. You can use this function to set up a TCP or UDP server.

Sound Management (Placeholder)

SpiLib includes a placeholder function for handling sound:

  • spilib_play_sound(sound_file)

    Plays the sound file specified by sound_file. (This requires integrating an audio engine.)

Random Numbers

To generate random numbers for game logic, such as AI decisions or level generation, you can use:

  • spilib_random_int(min, max)

    Generates a random integer between min and max (inclusive).
